Description

Our mightly little program can collect all this data, but what happens when the program terminates? All the data is lost! Let's fix this and add the possibility to save the data our program collects. This can be particularly useful for sending results to another computer or processing results in another program.

You should add a JButton to save files on a disk and also add a JTextField where the file path will be written.

You should structure the file in a way that each odd line contains a link and the line that follows it (even line) contains the corresponding title.

Also, let's update the way we've been creating the URLConnection to disguise as a human because some sites might send the wrong HTML code if they see that the query was sent by a bot. To achieve this, you should specify user-agent property as the property in the browser.

Use .setRequestProperty("User-Agent", "Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:63.0) Gecko/20100101 Firefox/63.0") for a URLConnection object. It's like Firefox running on Windows 10. This way, most sites will treat the bot as a human.

Objectives

In this stage, you should:

  1. Add a JTextField where the file path will be entered.
  2. Add a JButton to save the file.
  3. Set the names of the following components:
  • JTextField "UrlTextField";
  • JButton "RunButton";
  • JLabel "TitleLabel";
  • JTable "TitlesTable";
  • JTextField "ExportUrlTextField";
  • JButton "ExportButton".
