Description

In this final stage, we are going to complete our web crawler.

Create worker-threads that wait for new tasks in the task queue. Each task is a URL. If a thread-worker gets a task, it goes to the page, saves its title, collects all the links on the page, and adds these links as new tasks in the task queue.

The links and their titles will be searched breadth-first.

Don't forget to exclude the pages that have already been visited to avoid being redundant.

Now that we know we'll be needing thread-workers, there should be a way to specify the number of workers available for a task. Use a JTextField to accomplish this.

There should also be a way to keep track of the number of pages parsed: use a JLabel for this.

JLabel is also sufficient to keep track of the elapsed time when the crawling process begins.

Add two checkable restrictions:

  • Maximum crawling depth: if enabled, workers won't go too deep into the Web. For example, if "Max Depth" is set to 3, your program should parse the initial link (depth 1), all the links in the initial link (depth 2), and all the links in the pages opened from the initial link (depth 3), and that's it. The image below will help you visualize the workflow.
  • Time limit: if enabled, workers won't add tasks after the given time.

Finally, replace the existing JButton that starts the crawling process with a JToggleButton to Run/Stop the process. During the process, the button should be in its selected state. If any of the restrictions above become true or the tasks run out, the button should return to its default state (unselected). Use the setSelected(boolean b) method of the button to accomplish this.

Objectives

In this stage, you should:

  1. Replace the JButton with a JToggleButton.
  2. Add a JTextField to the window for specifying the number of thread-workers to be created.
  3. Add a JTextField to specify the maximum depth.
  4. Add a JCheckBox to enable the maximum depth restriction.
  5. Add a JTextField to specify the time limit.
  6. Add a JCheckBox to enable the time limit restriction.
  7. Add a JLabel to keep track of the number of parsed pages.
  8. Add a JLabel to keep track of the time elapsed after parsing begins.
  9. Set the names of the following components :
  • JTextField "UrlTextField";
  • JToggleButton "RunButton";
  • JTextField "DepthTextField";
  • JCheckBox "DepthCheckBox";
  • JLabel "ParsedLabel";
  • JTextField "ExportUrlTextField";
  • JButton "ExportButton".
